Flows
① Login (HTTP REST)
User
Types username + password
click / Enter
LoginWidget
Validates fields, disables button
onLoginClicked()
login(user, pass)
AuthManager
Builds JSON payload
{username, password[, email]}
POST /login
Server
Authenticates user
HTTP 200 + token
or HTTP 4xx + message
Response parsed in AuthManager → lambda callback:
✓ token present → emit loginSucceeded(username, token) → LoginWidget re-enables → emits loginSuccessful → MainWindow::onLoginSuccessful stores token, calls wsClient→connectWithToken(token), switches to chat view.
✗ error / missing token → emit loginFailed(message) → LoginWidget shows error label, re-enables button.

③ Initial Data — Channels, Contacts, Invitations
Server
Sends INITIAL_DATA unprompted
{channels[], contacts[], invitations[]}
JSON ← server
WebSocketClient
Parses InitialDataResponse, emit initialDataReceived(data)
Qt signal
MainWindow
onInitialDataReceived()
calls 3 sub-routines:
populateChannels()
ChannelPanel
setChannels() → builds QListWidget rows (ChannelRowWidget)
title, preview, unread badge, isGroup icon
Simultaneously:
→ cacheKnownUsers(data) — populates userNamesById map from contacts + channel participants + sets id 1 = "System"
→ userHomeWidget→setIncomingInvitations(data.invitations) — builds Accept/Reject rows in the UserHomeWidget tab
→ channelTitles map filled for quick lookup
④ Channel Selection — Load History
User
Clicks a channel row
currentItemChanged
ChannelPanel
emit channelSelected(channelId, title)
Qt signal
MainWindow
onChannelSelected() — stores currentChannelId, shows rightPanel, clears messages
openChannel(id)
WebSocketClient
Sends REQUEST_CHANNEL_HISTORY
{id_channel, before:0, limit:50}
JSON → server
Server
Returns CHANNEL_HISTORY
{messages[], has_more}
JSON ← server
WebSocketClient
Parses → emit channelHistoryReceived(history)
Qt signal
MainWindow
onChannelHistoryReceived() — sorts by id_message, renders each → RightPanel
+ calls markAsRead + clears unread badge
Each message is rendered via createMessageWidget(msg):
→ resolves id_sender → username via userNamesById
→ system messages: resolveAtMentions() replaces @ID with @username
→ creates MessageItemWidget (own messages left-aligned, others right-aligned)
→ added via rightPanel→addMessageWidget(widget) — auto-scrolls to bottom
⑤ Send Message
User
Types text, presses Send / Enter
emit sendRequested(text)
MainWindow
onSendMessageRequested(text) — validates currentChannelId
sendMessage(channelId, text)
WebSocketClient
Builds SEND_MESSAGE JSON, sends over socket
{type, id_channel, body}
JSON → server
Server
Persists, broadcasts NEW_MESSAGE to all members
⚠️ Note: After sending, MainWindow fires a 120ms fallback timer calling openChannel(currentChannelId) again — re-fetches history as a safety net in case the NEW_MESSAGE event is missed. The message will appear via flow ⑥ if properly received, or via this re-fetch otherwise.
⑥ Receive New Message (real-time push)
Server
Pushes NEW_MESSAGE to all channel members
{type, id_channel, message{…}}
JSON ← server
WebSocketClient
Parses SendMessageResponse, emit newMessageReceived(msg)
Qt signal
MainWindow
onNewMessageReceived()
always updates channel preview
updateChannelOnNewMessage()
ChannelPanel
Rebuilds ChannelRowWidget with new preview + incremented badge (if not active channel)
If msg.id_channel == currentChannelId: → appendMessageToView() → new MessageItemWidget appended to RightPanel, auto-scroll
If different channel & not from self: → badge incremented on the channel row only (user is notified visually)
⑦ Mark as Read — Unread Count Sync
MainWindow
Called at end of onChannelHistoryReceived
lastMessageId from sorted history
markAsRead(channelId, lastId)
WebSocketClient
Sends MARK_AS_READ
{id_channel, last_id_message}
JSON → server
Server
Updates read state, echoes back MARK_AS_READ with confirmed unread_count
JSON ← server
WebSocketClient
Parses MarkAsRead, emit updateChannelUnreadCount(id, count, lastId)
Qt signal
MainWindow
onUpdateChannelUnreadCount() → channelPanel→updateChannelUnreadCount()
Rebuilds row widget, badge updated / cleared
The client also immediately zeroes the local badge when the channel history loads (optimistic clear), without waiting for the server echo.
⑧ Invitations — Accept / Reject
User
Opens portrait → UserHomeWidget shows Invitations tab
userHomeRequested signal
MainWindow
hides rightPanel, shows userHomeWidget
clicks Accept/Reject
UserHomeWidget
emit acceptInvitationRequested(id) or rejectInvitationRequested(id)
Qt signal → lambda
WebSocketClient
acceptInvitation() or rejectInvitation()
{type:ACCEPT/REJECT_INVITATION, id_channel}
JSON → server
Server
Processes. Presumably sends updated INITIAL_DATA or channel added event.
⚠️ Incomplete flow: There is no handler for the server's response to accept/reject. The client sends the action but the UI (invitation list, channel panel) is not updated automatically — the user would need to reconnect to see the change.
⑨ Automatic Reconnection
Server / Network
WS connection drops
disconnected signal
WebSocketClient
onDisconnected() → emit disconnected() → startReconnectTimer()
Exponential backoff: 1s, 2s, 4s, 8s, 16s (max 5 retries)
Qt signal
MainWindow
onWsDisconnected() → disables UI, shows dialog
timer fires → attempt
WebSocketClient
onReconnectTimer() → socket.open(wsUrl)
On success: resets counter, triggers auth flow (→ ②)
if 5 retries fail
WebSocketClient
emit errorReceived("RECONNECT_FAILED", …)
